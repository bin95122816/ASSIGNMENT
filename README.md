# 📌 ASSIGNMENT

멘토링 과제 제출용 레포지토리입니다.

---

## 📚 과제 제출 방식

1. 해당 레포지토리를 Fork 합니다.
2. 본인 레포지토리를 Clone 합니다.
3. 본인 브랜치를 생성합니다. (브랜치 1개만 사용)
4. 과제 코드를 작성 후 Commit & Push 합니다.
5. Pull Request(PR)를 생성합니다.

* 1, 2, 3번은 한 번만 하고, 이후 과제 제출할 때는 4, 5번만 수행하면 됩니다.

---


---

## 📌 파일 작성 규칙

- 파일명은 문제 번호로 작성 (10828.c, 9012.c 등)
- 반드시 C로 작성
- 한 주차당 하나의 폴더 생성 (week1, week2 ...)

---

## 📌 과제 규칙

- 매주 백준 문제 2문제 필수
- 금요일까지 제출
- 책 코드 사용 가능 (단, 반드시 이해할 것)

---

## 📌 중요

- 코드 제출 후, 다음 멘토링 시간에 코드 설명이 진행됩니다.
- 단순 제출이 아니라, 코드의 동작 원리를 설명할 수 있어야 합니다.

---

## ❗ 주의사항

- 복사/붙여넣기만 하고 이해하지 않은 경우, 설명 과정에서 확인됩니다.
- 완벽한 정답보다 문제 해결 과정과 이해가 더 중요합니다.
